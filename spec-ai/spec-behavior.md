# 產品行為規格 (Behavior Spec)

## Spec

### 1. Program 頁面：動作輸入
* **入口**：採用日曆 (Calendar) 方式讓使用者選擇日期。
  * 支援切換時間週期檢視（日、週、月、季、年）。
  * 點選特定日期後，可選擇動作、輸入組次數與重量，以紀錄訓練。
* **預設動作範本機制**：
  * 提供預設的動作範本，讓使用者點選後產生範本，再自行修改實際的組數、次數與重量。
* **動作與器材組合清單**：
  * 理想上提供涵蓋訓練動作與器材組合的清單（例如：槓鈴深蹲、SSB 後腳抬高蹲），供選取後填寫紀錄。

### 2. 首頁 (Home Page)
* **Performance Overview 圖表**：
  * **最佳紀錄 (PR / 3RM / 5RM) 顯示**：
    * 系統由使用者的訓練紀錄（包含組數、次數、重量）中，自動篩選找出各反覆次數的最高紀錄。
    * 例如：3RM 是從該使用者所有「次數為 3」的紀錄中尋找最高重量，以此類推。
  * **動態 Tab 顯示邏輯**：
    * PR / 3RM / 5RM 等 Tab 需根據「使用者是否曾有該反覆次數的訓練紀錄」來決定是否出現。
    * 例如：如果沒有 1RM 紀錄，但有 10 筆 3 次的 Squat 紀錄，則僅顯示 3RM Tab，不顯示 1RM Tab。
* **自訂要追蹤 / 顯示的動作圖表**：
  * 有被使用者設定為「追蹤」的動作，方會顯示於首頁總覽圖表中。
  * **預設行為**：預設隨機挑選 3 個「已有訓練紀錄」的動作顯示（若未滿 3 個，則有幾個顯示幾個）。
  * **動作選項顯示條件**：必須要有紀錄，且至少包含一筆 3次 / 5次 / 1次 的紀錄才能出現在設定清單中，否則不顯示讓使用者選擇。
  * **MVP 限制**：最多可選擇 **三種** 動作進行追蹤與顯示。
  * **趨勢狀態指示**：各動作圖表上方需顯示該動作的表現趨勢狀態，包含：`↗ Trending Up`, `↘ Trending Down`, `Stable（尬電，你真的有在練嗎）`。
* **Body Fat Percentage 圖表**：
  * **自訂顯示開關**：允許使用者自行決定是否要在首頁追蹤 / 顯示體脂率圖表。
  * **啟用條件**：至少需有一筆體脂率紀錄，否則該選項 Disabled。
  * **時間與單位控制**：圖表需能切換單位，且支援左右拖拉時間軸。
  * **趨勢狀態指示**：圖表上方需顯示體脂率狀態，包含：`↗ Trending Up`, `↘ Trending Down`, `Stable`。
  * **UI 風格**：狀態顯示之風格與文字需與 Performance Overview 保持一致。

## Need to Confirm

* **[使用者提出] 動作自定義輸入 (MVP 版本)**：雖然理想上有完整的「訓練動作 + 器材」清單供選擇，但動作變化太多，MVP 版本需確認是否先讓使用者「自己輸入動作名稱」即可？
* **[使用者提出] 「自訂追蹤動作」功能入口與呈現**：讓使用者選擇「要追蹤/顯示之動作」的功能該擺在哪一頁？UI 如何呈現才能保持簡約，並符合 Apple Human Interface Guidelines？
* **[使用者提出] 首頁追蹤圖表 UI 優化**：目前前端已有表現追蹤圖表的初步實作，需檢視並討論如何調整才能更簡約、更貼近 Apple Human Interface Guidelines 規範。
* **[使用者提出] 「自訂追蹤體脂率」功能入口與呈現**：讓使用者決定是否顯示體脂率的設定或開關，該擺在哪一頁？UI 該如何設計以符合 Apple Human Interface Guidelines？
* **[使用者與AI共同提出] 體脂率圖表合併評估確認**：既然 Performance Overview 其實已將訓練表現與體脂趨勢放在同一張圖表內，是否還需要將「Body Fat Percentage 圖表」獨立成另一個專屬顯示區塊？
