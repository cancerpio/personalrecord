# 產品行為規格 (Behavior Spec)

## Spec

### 1. Program 頁面：動作輸入
* **入口**：採用日曆 (Calendar) 方式讓使用者選擇日期。
  * 支援切換時間週期檢視（日、週、月、季、年）。
  * 點選特定日期後，可選擇動作、輸入組次數與重量，以紀錄訓練。
* **預設動作範本機制**：
  * 提供預設的動作範本，讓使用者點選後產生範本，再自行修改實際的組數、次數與重量。
* **動作與器材組合清單 (MVP 混合模式)**：
  * **基礎預設清單**：系統內建最具代表性的核心動作（如：Squat 深蹲、Bench Press 臥推、Deadlift 硬舉...等）做為預設下拉選單。
  * **支援自訂新增**：使用者可在選單中選擇「新增自訂動作」，自行輸入未涵蓋的動作名稱（例如：SSB 後腳抬高蹲）。
  * **個人化紀錄儲存**：自訂動作一旦儲存，會寫入使用者的專屬選單歷史中。未來新增紀錄時可直接選取，避免大小寫或選字不同而造成的資料不一致，確保 PR 追蹤與趨勢圖表計算正確。

### 2. 首頁 (Home Page)
* **廣度綜覽 (Overview - Sparkline Pills)**：
  * **位置與功能**：位於首頁最上方，提供輕量、快速的多動作表現總覽。
  * **動態產生**：系統會動態掃描所有的歷史紀錄，自動提取使用者「有練過」的獨特動作，並挑選最近的 3 個顯示在此區域。
  * **連莊計算 (Streak Calculation)**：與傳統迷你折線圖不同，此區塊採用簡約的「狀態膠囊 (Status Pill)」。系統會從最新的一筆資料往前推算，比對連續上升、連續下降或連續持平的趨勢，並呈現天數（例如：`↗ 連續 3 天上升` 或 `→ 連續 2 天停滯`）。
  * **目的**：讓使用者不需解讀圖表，只要看一眼膠囊顏色和文字，即可迅速掌握近期主要訓練動作的連莊進退步狀態。
* **深度深掘 (Deep Dive - 統一疊合主圖表 / Hero Section)**：
  * **位置與地位**：位於 Sparklines 下方，為主視覺區塊 (Hero Section)。首頁僅保留此唯一一張大型詳細圖表，極致符合 Apple HIG 資訊集中的簡約精神。
  * **單一動作專注**：圖表一次僅顯示「單一動作」的表現趨勢，可透過圖表上方的選單（如：`Squat ▾`）快速切換不同的動作。
  * **雙 Y 軸疊合顯示 (Dual Y-Axis)**：
    * **左側 Y 軸**：顯示訓練重量（公斤/磅），繪製該動作的 PR / 3RM / 5RM 趨勢。
    * **右側 Y 軸**：顯示體脂肪率（百分比 %）。
  * **體脂疊加開關 (Body Fat Overlay)**：圖表區塊提供一個開關（Toggle），點擊後即可在當前的重量趨勢圖上，疊加上體脂肪率的曲線。讓使用者能在一張圖內直觀對比「體脂/體重變化」對「力量表現」的關聯影響。若無體脂資料，該開關將 Disable。
  * **最佳紀錄顯示**：同原設定，系統會顯示該動作各次數的「最新紀錄」與「歷史最高紀錄」。

### 3. 資料儲存與驗證模式
系統需支援三種資料儲存與驗證模式切換，以適應不同執行環境：
* **Client 端儲存 (Local 模式)**：
  * 資料純粹存放在使用者裝置的瀏覽器中（LocalStorage）。
  * 不進行任何遠端連線或身分驗證，適合無網路或純單機測試使用。
* **Backend 端儲存 - LINE Mini App 驗證 (LIFF 模式)**：
  * 在 LINE 應用程式內執行時的模式。
  * 透過 LINE 官方提供的驗證機制 (`liff.getIDToken()`) 確認使用者身分。
  * 確保在 LINE 生態系中能與使用者的 LINE 帳號緊密綁定並獲得安全的資料存取。

### 4. 全域資料同步 (Global Data Synchronization)
* **單一資料來源 (Single Source of Truth) 行為**：
  * 當使用者在 **Program 頁面** 成功儲存一筆訓練紀錄時，系統不會要求使用者手動重整網頁。
  * 該筆新紀錄會立即寫入全域狀態管理中心 (Pinia Store)。
  * **Dashboard 首頁圖表** (以及未來的 Log 歷史頁面) 會因為訂閱了該狀態中心，而在儲存瞬間「即時 (Reactively)」重新渲染並畫出新資料的點位。
* **無縫切換體驗**：
  * 在任何網路不穩定或是切換 `Local` / `LIFF` 驗證模式的情境下，如果遇到資料寫入延遲，UI 應呈現非同步的 Loading 狀態，確保使用者明確知道資料正在同步中，才不會誤以為紀錄失敗。
